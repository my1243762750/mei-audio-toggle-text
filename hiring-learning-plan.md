# 语音/AI API 岗位需要什么

## 他们真正需要的能力

不是只会“调用 API”，而是能把 TTS/STT/API 能力做成稳定可用的产品功能。

核心能力：

- 会接第三方 AI API：TTS、STT、LLM、文件上传、流式返回。
- 会做前后端联调：页面上传音频、输入文本、展示结果、播放音频。
- 会做后端接口：鉴权、参数校验、错误处理、超时、重试。
- 会处理文件：音频格式、图片/音频上传、临时文件、对象存储。
- 会控制成本：限制请求频率、限制文件大小、缓存结果、记录用量。
- 会做稳定性：日志、异常提示、失败重试、接口降级。
- 会理解业务：把“模型能力”包装成用户能用的功能。

## 对应到文字转语音和语音转文字

文字转语音：

- 用户输入文字。
- 后端调用 TTS API。
- 返回音频文件或音频 URL。
- 前端播放或下载音频。

语音转文字：

- 用户上传音频。
- 后端调用 STT API。
- 返回识别文本。
- 前端展示、编辑、复制文本。

## 学习目标

第一阶段：做通最小 Demo

- 文本输入框。
- 点击按钮生成语音。
- 上传音频文件。
- 点击按钮识别文字。
- 前后端能完整跑通。

第二阶段：做得像真实项目

- 加 loading、错误提示。
- 限制文件大小和文本长度。
- 保存历史记录。
- 音频结果可播放。
- 接口失败时给明确提示。

第三阶段：贴近企业要求

- 加用户鉴权。
- 加调用次数限制。
- 记录 API 消耗。
- 做日志和异常排查。
- 支持流式语音或长音频切分。
